# PawHandoff AI：AI 工作流与 Prompt 流程

## 1. 工作流总览

```text
主人结构化输入
  → 字段校验与最小化处理
  → 生成交接包
  → 主人确认/修订
  → 照护方读取每日任务并打卡
  → 完成项 + 照片数量/说明 + 事实备注
  → 生成今日报告与微信简报
  → 双方查看，异常时转人工沟通/专业兽医
```

AI 只负责信息整理、改写和缺项提示，不判断照片真伪、不诊断疾病，也不承诺照护安全。涉及异常时，输出应鼓励及时联系主人和专业兽医，不擅自给药或改变方案。

## 2. 交接包生成 Prompt

代码接口：`ai_generate_handoff_pack(inputs)`

**System Prompt（建议）**

```text
你是宠物照护交接信息整理助手。你的任务是把主人提供的事实整理得清楚、简洁、可执行。
不得补造宠物习惯、剂量、联系人或医疗结论；缺失信息必须标为“待主人确认”。
不要使用“监控”“偷拍”等表达。强调授权区域、最少必要照片、双方确认和隐私保护。
异常内容仅提供联系与升级流程，不替代兽医诊断，不承诺绝对安全。
使用中文 Markdown 输出。
```

**User Prompt 模板**

```text
请根据以下 JSON 生成：
1. 宠物照顾说明书
2. 每日任务清单（可勾选、动作明确）
3. 异常情况处理流程
4. 给照护方的简洁版说明
5. 主人出门前准备清单
6. 隐私与安全提醒

主人输入：{{inputs_json}}
```

**生成后校验**

- 不得创造输入中不存在的药名、剂量和时间。
- 联系人、允许/禁入区域、喂食量缺失时提示确认。
- 对猫/狗切换“猫砂”与“遛狗”措辞。
- 始终包含门窗确认、仅进入授权区域和不拍无关隐私的提醒。

## 3. 今日报告生成 Prompt

代码接口：`ai_generate_daily_report(checkin_data)`

**System Prompt（建议）**

```text
你是宠物照护记录整理助手。只总结打卡中可验证的事实，并明确区分“已完成”“未勾选”“照护方备注”。
未勾选不等于未完成，只能写“待确认”。照片只能作为辅助说明，不得宣称已验证真实性。
不要诊断疾病或淡化异常。发现异常或信息冲突时，建议联系主人，必要时咨询专业兽医。
最后生成一段自然、简短、适合微信发送的汇报。
```

**User Prompt 模板**

```text
根据以下打卡 JSON 生成：
1. 今日任务完成情况
2. 宠物状态摘要
3. 饮食、饮水、排泄情况
4. 是否有异常
5. 照片证据说明
6. 明天需要注意什么
7. 微信简洁汇报

打卡信息：{{checkin_json}}
```

## 4. 真实大模型替换位置

当前 `app.py` 使用规则驱动的 mock 文本生成。接入真实模型时保留两个函数签名，在函数内部完成：输入字段白名单 → 敏感信息最小化 → API 调用 → Markdown 结构校验 → 风险措辞校验 → 返回文本。

建议将 API Key 放在环境变量或 Streamlit secrets 中，不写入仓库；默认不把真实姓名、完整住址、门锁密码、未经处理的室内照片发送给第三方模型。产品化前还应增加用户授权、数据删除、访问控制、失败回退和人工确认机制。

## 5. MVP 的 mock 逻辑

- 交接包：把表单字段映射到六个固定章节，根据宠物类型调整清洁任务用词。
- 今日报告：按勾选项计算完成/待确认列表；备注或异常标记会触发“需关注”措辞。
- 微信文本：压缩为宠物名、完成项、状态、照片数、备注和明日动作。
- 任何输出均提示：本工具降低沟通成本和信任风险，不提供安全或医疗保证。
