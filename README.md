# PawHandoff AI

PawHandoff AI 是我正在设计的一款宠物照护交接工具，适用于主人旅行、出差、节假日回家，需要把宠物暂时托付给朋友、代喂人或寄养方的场景。

我想解决的不是“去哪里找代喂人”，而是找到人之后，怎样把宠物的习惯和照护要求交代清楚，怎样让每天的执行情况更容易同步，以及出现异常时怎样留下双方都看得懂的记录。

因此，PawHandoff AI 会把主人填写的信息整理成一份标准照护交接包，为照护方生成每日任务，并将打卡、照片和备注汇总成给主人看的今日报告。

## 产品思路

一次宠物托付通常涉及三类信息：

- **交接前**：吃什么、喂多少、宠物性格、空间边界和紧急联系人。
- **照护中**：今天完成了什么，进食、饮水、排泄和精神状态如何。
- **出现变化时**：观察到了什么、是否已联系主人、接下来应该关注什么。

这些内容经常散落在聊天记录和临时口头说明里。这个项目尝试把它们组织成一个简单、可重复的协作流程，而不是把照护关系变成对人的监控。

## 本地运行

建议使用 Python 3.10 或更高版本。在当前目录执行：

```bash
pip install -r requirements.txt
streamlit run app.py
```

浏览器通常会自动打开 `http://localhost:8501`。

## 当前功能

- 主人端表单：记录宠物习惯、喂养要求、空间边界、异常判断和紧急联系人。
- 一键生成：照顾说明书、每日任务、异常流程、照护方简版、出门准备清单及隐私提醒。
- 照护方打卡：完成项勾选、1–3 张照片上传、异常与补充备注。
- 今日报告：自动整理完成情况、宠物状态、饮食饮水排泄、异常、照片说明和微信简报。
- 二维码照护卡：展示本地概念二维码、宠物名与紧急联系人。
- Markdown 导出：下载交接包和今日报告。

## 使用流程

1. 进入“主人端生成交接包”，填写必填项并生成。
2. 切换到“代喂人 / 寄养方打卡”，勾选任务、上传图片并提交。
3. 在“今日报告”查看 AI 整理结果并下载 Markdown。
4. 在“二维码照护卡”查看概念卡片。

## 技术实现

- UI：Streamlit
- 二维码：qrcode + Pillow
- 状态：仅使用 `st.session_state`，刷新/关闭会话后可能丢失
- 文本生成：当前使用本地规则生成，不需要 API Key；`ai_generate_handoff_pack()` 和 `ai_generate_daily_report()` 保留了独立接口，方便后续接入大模型
- 照片：仅在当前 Streamlit 会话中用于计数和演示，不落库、不做识别

## 设计边界

- 不提供人员派单、真实支付或实时定位。
- 不把任务留痕描述成监控，也不鼓励拍摄无关的私人空间。
- 不保证完全消除照护风险，生成内容需要主人和照护方共同确认。
- 不提供疾病诊断，异常情况应及时联系主人，必要时咨询专业兽医。

## 后续计划

- 找真实养宠人与照护方试用，精简不必要的表单字段。
- 支持主人修改任务、按实际照护天数生成每日记录。
- 验证寄养店和上门喂养团队是否需要多人协作及历史报告。

## 项目结构

- `app.py`：Streamlit 应用与本地文本生成逻辑
- `prompt_flow.md`：文本生成流程、约束和模型替换方案
- `requirements.txt`：运行依赖
