# 项目名称

**PawHandoff AI：宠物代喂与寄养智能交接助手**

# 一句话介绍

将主人零散的宠物照护要求生成标准交接包和每日任务，并把照护方的打卡、照片与备注自动整理成今日报告，降低双方沟通成本与信任风险。

# 1. 机会判断

**目标用户：**节假日回家、出差、旅游或临时无法照顾宠物的猫狗主人；协作用户包括朋友代喂者、上门代喂人和寄养店。主人常因喂食、饮水、清洁、用药和宠物习惯交代不清而焦虑，也担心上门边界、门窗确认和寄养卫生安全；照护方则担心信息散落在聊天记录中、遗漏任务，出现异常后责任难以说明。现有平台多聚焦“找到照护人”，交接是否清楚、执行是否透明、异常是否留痕仍有工具机会。

**为什么现在：**养宠人群与节假日、旅行、异地学习工作场景持续存在，宠物服务逐渐线上化；生成式 AI 已能低成本地把自然语言整理成标准说明、任务和摘要。PawHandoff AI 不承诺完全保证宠物安全，而是用流程标准化、任务打卡和照片留痕降低误解与信任风险。

# 2. 用户证据

本阶段不编造市场数据，先验证真实行为与付费意愿：计划半结构化访谈 **5–10 位养宠人**，并访谈 **2–3 位朋友代喂者/职业代喂人/寄养店员工**。重点追问最近一次托付的完整过程、交代花了多久、最担心什么、出现过哪些遗漏、当前如何留痕、是否愿意让照护方使用打卡页及愿付价格。

同步在小红书/抖音搜索“代喂猫、宠物寄养、猫咪寄养踩雷、上门喂猫、节假日宠物寄养”等关键词，并观察淘宝相关服务评论；记录而非臆测评论中反复出现的：喂养遗漏、环境卫生、混养/应激、上门空间边界、门窗确认、照片真实性、异常联系和责任争议。使用匿名编码表保存链接、日期、场景和原话摘要，形成可回查的证据库。

# 3. MVP 样例

本地 Streamlit Demo 覆盖一个完整闭环：①主人填写宠物信息并生成照顾说明书、每日任务、异常流程、简版说明、出门清单和隐私提醒；②代喂人/寄养方勾选任务、上传 1–3 张照片并备注；③AI 自动生成今日照顾报告和微信简报；④二维码照护卡展示快速进入任务的概念；⑤交接包与日报均可导出 Markdown。当前使用 mock AI 与会话状态，不上传真实隐私、不接支付或定位。

# 4. 商业判断

**C 端付费者**是需要短期托付宠物的主人，付费原因是减少重复交代、让每日执行更清楚，并在异常时保留双方共同可读的记录。建议“基础交接包免费；完整交接包 **9.9 元/次**；多日打卡与报告 **19.9 元/次**”。**B 端客户**为寄养店、上门喂养团队和宠物店，价值是统一服务 SOP、减少客服沟通并形成更专业的客户报告，建议 SaaS **99–299 元/月**，按门店/成员数分档。

获客优先从高意图场景切入：小红书养宠内容与模板分享、校园养宠社群、宠物店/寄养店联合试用，以及春节、国庆、寒暑假前的节点内容投放。早期先验证使用与付费，不做大规模投放。

# 5. 2周验证计划

**第 1 周：问题与 Demo 验证。**完成 5–10 位主人及 2–3 位照护方访谈；整理公开社群/评论证据；让受访者现场完成 Demo，记录填表耗时、看不懂或不愿填写的字段，并测试“基础免费 / 9.9 / 19.9”三档选择。资源：1 名产品开发者、访谈激励约 300–500 元、可用的手机和本地 Demo。

**第 2 周：真实任务试用。**招募 5 组近期确有代喂/寄养需求的主人—照护方配对，连续使用 2–3 天；观察交接包是否真的发出、照护方是否每日打卡、主人是否阅读报告。核心判定：≥4 组完成首次交接，≥3 组完成连续打卡，≥3 位主人认为比微信散聊更清楚，且至少 2 位愿以 9.9/19.9 元继续使用；同时记录照护方是否愿意配合及任何隐私顾虑。未达标则根据最大阻力调整字段、报告或转向 B 端 SOP 工具，再进行下一轮验证。
